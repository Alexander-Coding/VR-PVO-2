=== M60: объединение и выброс гильз ===

1) ОБЪЕДИНИТЬ m60 И m60-1
   - Остановите Play mode.
   - В Hierarchy: перетащите объект "m60-1" на "m60" (сделать m60-1 дочерним к m60).
   - Так оба ствола будут двигаться как один объект.

2) ГИЛЬЗЫ (m60-case)
   - Скрипт поддерживает и префаб, и объект из сцены. В поле Shell Prefab можно перетащить объект m60-case из Hierarchy — гильзы будут клонироваться из него.
   - Либо оставьте Shell Prefab пустым: скрипт найдёт объект по имени "m60-case" в сцене и будет использовать его как образец.

3) НАСТРОЙКА СКРИПТА M60ShellEjection
   - Скрипт вешается на объект "m60" (Add Component → M60 Shell Ejection).

   ЧТО ЗНАЧАТ ПОЛЯ В INSPECTOR:
   • Ejection Point 1 и 2 — это точки, откуда вылетают гильзы (обычно деталь "m60 receiver" у каждого ствола).
     Если оставить их пустыми (None), скрипт сам найдёт при старте сцены:
     - точка 1: дочерний объект "m60 receiver" у m60;
     - точка 2: объект "m60 receiver" внутри "m60-1" (второй ствол).

   КАК НАЗНАЧИТЬ ВРУЧНУЮ (если автопоиск не сработал):
   - В Hierarchy раскройте m60 (стрелка слева от имени).
   - Найдите объект "m60 receiver" — перетащите его в поле "Ejection Point 1" в Inspector.
   - Раскройте "m60-1", внутри найдите "m60 receiver" — перетащите его в "Ejection Point 2".
   - В "Shell Prefab" можно перетащить объект m60-case из сцены или оставить пустым (скрипт найдёт m60-case по имени).

   Остальные поля (Ejection Force, Fire Interval и т.д.) — по желанию.

4) КАК РАБОТАЕТ
   - В редакторе: удержание Пробела или ЛКМ — тестовая стрельба, гильзы вылетают из обоих стволов.
   - Для VR/кнопки: вызывайте из своего кода M60ShellEjection.Fire() при нажатии на спуск (или привязайте к Animation Event).

5) ЕСЛИ ГИЛЬЗЫ НЕ ВЫЛЕТАЮТ
   - Убедитесь, что m60-1 является дочерним к m60 и внутри есть объект "m60 receiver".
   - Либо создайте два пустых объекта под m60 (ПКМ → Create Empty): "Ejection1" и "Ejection2", поставьте их у выходов гильз и перетащите в Ejection Point 1 и 2 в Inspector.
